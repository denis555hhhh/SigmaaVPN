🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ:
═════════════════════════════════════════════════════════════════════════════

Google Sign-In Flow:
  1. Пользователь нажимает кнопку "Войти через Google"
  2. Инициализируется Google Sign-In с Client ID
  3. Открывается окно выбора аккаунта Google
  4. Google возвращает JWT токен
  5. Код декодирует токен (base64 decode)
  6. Извлекаются данные: name, email, sub (Google ID), picture
  7. Отправляется POST запрос на /api/register-google или /api/login-google
  8. Сервер создает пользователя или находит существующего
  9. Создается подписка (для регистрации)
  10. Данные сохраняются в localStorage
  11. Пользователь перенаправляется на главную страницу

API Endpoints:
  POST /api/register-google
    Input: { username, email, google_id, picture }
    Output: { успех, user_id, username }
    
  POST /api/login-google
    Input: { email, google_id }
    Output: { успех, user_id, username, email }
