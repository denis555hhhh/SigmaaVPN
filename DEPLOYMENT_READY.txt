STEP 2: Authenticate with GitHub
─────────────────────────────────────────────────────────────────────────────
When prompted, enter:
  Username: deni555hhhh
  Password: Your GitHub Personal Access Token (or password)

To create a token:
  1. Go to https://github.com/settings/tokens
  2. Click "Generate new token"
  3. Select scopes: repo, workflow
  4. Copy the token and use it as password

STEP 3: Deploy on Railway
─────────────────────────────────────────────────────────────────────────────
After code is pushed to GitHub:

1. Go to https://railway.app
2. Sign in with GitHub
3. Click "New Project"
4. Select "Deploy from GitHub"
5. Choose repository "sigma"
6. Click "Deploy"

Wait 2-5 minutes for Railway to build and deploy your site.

2. Railway Deployment:
   - Railway will automatically detect Procfile
   - It will install dependencies from requirements.txt
   - Your app will be deployed to a public URL

   - SQLite database is included (sigmavpn.db)
   - Railway will use it automatically
   - Data persists between deployments

   - app.py reads these variables
   - No manual configuration needed

   - After deployment, update API_URL in JavaScript files
   - Change from: http://localhost:5000
   - Change to: https://sigma-production.railway.app
   - Then commit and push again
