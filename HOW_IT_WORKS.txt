ЭТАП 1: ПОСЕЩЕНИЕ САЙТА
├─ Пользователь открывает https://sigmavpnn-production.up.railway.app

💳 ПЛАТЕЖНАЯ СИСТЕМА (Yoomoney):

ПРОЦЕСС:
1. Бот генерирует ссылку на платёж
   └─ https://yoomoney.ru/quickpay/confirm.xml?receiver=4100118775331265&...

2. Пользователь открывает ссылку
   └─ Видит форму платежа Yoomoney

3. Пользователь вводит данные карты
   └─ Выбирает способ оплаты

4. Yoomoney обрабатывает платёж
   └─ Проверяет данные карты

5. Платёж подтверждается
   └─ Деньги поступают на кошелёк

6. Бот получает уведомление
   └─ Активирует подписку

ОСОБЕННОСТИ:
├─ Вечные ссылки (никогда не истекают)
├─ Комиссия 2%
├─ Быстрая обработка
├─ Безопасность
└─ Поддержка всех способов оплаты

💾 БД (SQLite):

ТАБЛИЦЫ:

1. telegram_users
   ├─ id (PRIMARY KEY)
   ├─ telegram_id (UNIQUE)
   ├─ username
   ├─ first_name
   └─ created_at

2. telegram_payments
   ├─ id (PRIMARY KEY)
   ├─ telegram_id (FOREIGN KEY)
   ├─ plan (Базовая/Стандарт/Премиум)
   ├─ amount (сумма в рублях)
   ├─ status (pending/completed)
   ├─ payment_id (UNIQUE)
   └─ created_at

3. telegram_subscriptions
   ├─ id (PRIMARY KEY)
   ├─ telegram_id (FOREIGN KEY)
   ├─ plan (Базовая/Стандарт/Премиум)
   ├─ status (active/expired)
   ├─ start_date
   ├─ end_date
   ├─ devices (количество устройств)
   └─ FOREIGN KEY (telegram_id)

СИНХРОНИЗАЦИЯ:
├─ Веб-сайт читает из БД
├─ Бот пишет в БД
├─ Панель управления читает из БД
└─ Все компоненты используют одну БД
