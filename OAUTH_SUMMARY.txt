📊 ТЕХНИЧЕСКИЕ ДЕТАЛИ:
═════════════════════════════════════════════════════════════════════════════

Google Sign-In Flow:
  1. Пользователь нажимает кнопку "Войти через Google"
  2. register.js/login.js загружают Client ID из oauth_config.json
  3. Инициализируется Google Sign-In с Client ID
  4. Открывается окно выбора аккаунта Google
  5. Google возвращает JWT токен
  6. Код декодирует токен (base64 decode)
  7. Извлекаются данные: name, email, sub (Google ID), picture
  8. Отправляется POST запрос на /api/register-google или /api/login-google
  9. Сервер создает пользователя или находит существующего
  10. Создается подписка (для регистрации)
  11. Данные сохраняются в localStorage
  12. Пользователь перенаправляется на главную страницу

API Endpoints:
  POST /api/register-google
    Input: { username, email, google_id, picture }
    Output: { успех, user_id, username }
    
  POST /api/login-google
    Input: { email, google_id }
    Output: { успех, user_id, username, email }
