🚀 КАК ИСПОЛЬЗОВАТЬ
═══════════════════════════════════════════════════════════════════════════════

Способ 1: Автоматическая синхронизация (рекомендуется)
───────────────────────────────────────────────────────────────────────────────

1. Откройте папку проекта
2. Дважды кликните на START.bat
3. Автоматически произойдет:
   ├─ Синхронизация БД с Railway
   ├─ Запуск Flask сервера
   └─ Открытие браузера с панелью
4. Панель показывает актуальные данные с Railway


Способ 2: Ручная синхронизация
───────────────────────────────────────────────────────────────────────────────

1. Откройте Command Prompt
2. Перейдите в папку проекта
3. Выполните: python sync_db_from_railway.py
4. Затем запустите: python admin_server.py
5. Откройте браузер: http://localhost:5000


⚙️  НАСТРОЙКА RAILWAY CLI (ОПЦИОНАЛЬНО)
═══════════════════════════════════════════════════════════════════════════════

Для полной синхронизации с Railway установите Railway CLI:

Шаг 1: Установить Railway CLI
  npm install -g @railway/cli

Шаг 2: Авторизоваться
  railway login

Шаг 3: Выбрать проект
  railway link

Шаг 4: Готово!
  Теперь START.bat будет синхронизировать БД с Railway


📊 ПРОЦЕСС СИНХРОНИЗАЦИИ
═══════════════════════════════════════════════════════════════════════════════

При запуске START.bat происходит:

1️⃣ Проверка Python и зависимостей
   └─ Установка Flask если нужно

2️⃣ Синхронизация БД
   ├─ Создание резервной копии (sigmavpn.db.backup)
   ├─ Скачивание БД с Railway
   ├─ Проверка целостности БД
   └─ Использование локальной БД если Railway недоступна

3️⃣ Запуск Flask сервера
   ├─ Подключение к синхронизированной БД
   ├─ Запуск API endpoints
   └─ Запуск на http://localhost:5000

4️⃣ Открытие браузера
   └─ Панель управления с актуальными данными

❌ ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ
═══════════════════════════════════════════════════════════════════════════════

Проблема: "Railway CLI не установлен"
Решение: npm install -g @railway/cli

Проблема: "Не удалось подключиться к Railway"
Решение: railway login (авторизуйтесь в Railway)

Проблема: "Используется локальная БД"
Решение: Это нормально, если Railway недоступна

Проблема: "Синхронизация не работает"
Решение: Проверьте что Railway CLI установлен и авторизован
