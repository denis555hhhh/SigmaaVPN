⚡ ВАРИАНТ 1: ТЕСТИРОВАНИЕ API (БЕЗ БРАУЗЕРА):
═════════════════════════════════════════════════════════════════════════════

1. Откройте командную строку
2. Запустите: python test_google.py
3. Смотрите результаты тестов

Это протестирует:
  ✓ Регистрацию через Google
  ✓ Вход через Google
  ✓ Обычную регистрацию
  ✓ Обычный вход

⚡ ВАРИАНТ 2: ТЕСТИРОВАНИЕ В БРАУЗЕРЕ (С ВРЕМЕННЫМ CLIENT ID):
═════════════════════════════════════════════════════════════════════════════

1. Откройте register.js
2. Найдите строку:
   client_id: '1234567890-abcdefghijklmnopqrstuvwxyz.apps.googleusercontent.com'

3. Замените на любой тестовый ID (например):
   client_id: 'test-client-id-12345.apps.googleusercontent.com'

4. Запустите simple_start.bat
5. Откройте http://localhost:8080/register.html
6. Нажмите "Войти через Google"

ПРИМЕЧАНИЕ: Без реального Client ID кнопка не будет работать,
но вы сможете видеть логи в консоли браузера (F12).

⚡ ВАРИАНТ 3: ПОЛНОЕ ТЕСТИРОВАНИЕ (С РЕАЛЬНЫМ CLIENT ID):
═════════════════════════════════════════════════════════════════════════════

1. Откройте GOOGLE_SETUP.txt
2. Следуйте инструкциям по получению Google Client ID
3. Обновите Client ID в register.js и login.js
4. Запустите START_WITH_GOOGLE.bat
5. Тестируйте Google Sign-In в браузере

📊 ПРОВЕРКА ЛОГОВ:
═════════════════════════════════════════════════════════════════════════════

Откройте консоль браузера (F12 → Console) и смотрите логи:

✓ "📤 Отправка запроса регистрации Google..."

Если видите эти логи - всё работает!

🔍 ПРОВЕРКА БД:
═════════════════════════════════════════════════════════════════════════════

После регистрации через Google проверьте БД:

1. Запустите: python db_launcher.py
2. Откройте админ-панель
3. Перейдите в "Пользователи"
4. Найдите пользователя с Google регистрацией

Должны быть:
  ✓ Email пользователя
  ✓ Username
  ✓ Пароль начинается с "google_"

❌ ЕСЛИ НЕ РАБОТАЕТ:
═════════════════════════════════════════════════════════════════════════════

1. Проверьте что API сервер запущен:
   - Откройте http://localhost:5000/api/stats
   - Должен вернуть JSON

2. Проверьте логи в консоли браузера (F12):
   - Ищите ошибки
   - Смотрите сообщения об ошибках

3. Проверьте логи API сервера:
   - Смотрите окно с python app.py
   - Ищите ошибки

4. Запустите test_google.py:
   - python test_google.py
   - Смотрите результаты

5. Прочитайте GOOGLE_SETUP.txt:
   - Там есть решения типичных проблем

РЕКОМЕНДУЕМЫЙ ПОРЯДОК ДЕЙСТВИЙ:

1. Запустите test_google.py (проверка API)
2. Запустите simple_start.bat (запуск серверов)
3. Откройте http://localhost:8080/register.html
4. Откройте консоль браузера (F12)
5. Попробуйте обычную регистрацию (без Google)
6. Если обычная регистрация работает - переходите к Google
7. Получите Google Client ID (GOOGLE_SETUP.txt)
8. Обновите Client ID в коде
9. Тестируйте Google Sign-In
